# reasoning-filter

Can a prompt reliably separate original reasoning from AI slop — even after the slop has been "humanized"?

## Thesis

AI slop has a fingerprint: high fluency, low information density, zero novel claims. Every sentence could be swapped with another and nothing changes.

Original reasoning has the opposite fingerprint: specific, falsifiable claims that couldn't have been generated by a generic prompt. The claims depend on each other — remove one and the argument breaks.

This difference is computable. A scoring prompt can measure it. And it's robust against surface-level "humanizer" tools that strip AI writing tics.

## Results

| Variant | Score |
|---------|-------|
| Original posts | **8.75** |
| Raw AI slop | 3.75 |
| Humanized AI slop | 2.40 |

The humanizer *lowered* scores. Stripping rhetorical devices from empty prose exposes the skeleton.

**Argument Dependency Chain** is the strongest discriminator: originals score 9.0, humanized slop scores 1.0. You can't fake an argument chain by editing commas. You'd have to actually think.

Full results: [`experiment/RESULTS.md`](experiment/RESULTS.md)

## Reproduce It

### 1. Understand the scoring prompt

[`experiment/scoring_prompt.md`](experiment/scoring_prompt.md) defines six dimensions:

| Dimension | Measures |
|-----------|----------|
| Falsifiable Claim Density (FCD) | Statements that could be proven wrong |
| Novel Concept Introduction (NCI) | Ideas not findable via generic search |
| Argument Dependency Chain (ADC) | Longest chain of interdependent paragraphs |
| Specificity Ratio (SR) | Named numbers, entities, mechanisms vs. generic claims |
| Interchangeability Index (II) | Sentences that could be swapped into another article |
| Hedging Frequency (HF) | "It remains to be seen", "arguably", etc. |

### 2. Read the samples

All six texts are in [`experiment/samples/`](experiment/samples/):

- `original_a_last_signal.md` — from the [Vector Space](https://kimjune01.github.io/vector-space) blog series
- `original_b_keywords_tiny_circles.md` — from the same series
- `slop_a_trust_signals.md` — AI-generated on the same topic as Original A
- `slop_b_semantic_advertising.md` — AI-generated on the same topic as Original B
- `humanized_a_trust_signals.md` — Slop A after running through [blader/humanizer](https://github.com/blader/humanizer)
- `humanized_b_semantic_advertising.md` — Slop B after the same humanizer

### 3. Run the scorer

Feed the scoring prompt + any sample text to Claude (or any frontier LLM). Compare scores across variants.

The slop was generated using the prompts in [`experiment/slop_generation_prompts.md`](experiment/slop_generation_prompts.md). The humanizer skill is documented in [`experiment/humanizer_SKILL.md`](experiment/humanizer_SKILL.md).

### 4. Try your own texts

The scoring prompt is model-agnostic. Feed it any text and see where it lands.

## Why this matters

Blog posts are becoming public specs. The people publishing original technical reasoning are writing the most valuable inbound emails in any inbox. Current email agents filter by urgency and sender reputation, missing this signal entirely.

An inbox agent that scores inbound by *density of original reasoning* would instantly separate high-signal messages from thought-leadership spam — and no humanizer can fool it, because humanizers operate on surface patterns, not structural reasoning.

## Disclaimer

This entire experiment — the scoring prompt, the slop generation, the humanization, the scoring runs, and this README — was conducted by [Claude Opus 4.6](https://docs.anthropic.com/en/docs/about-claude/models) via [Claude Code](https://claude.ai/claude-code), directed by a human ([kimjune01](https://github.com/kimjune01)) who provided the thesis and steered the experimental design.

That means Claude scored texts that Claude also generated. This is a limitation. The scorer may have systematic biases toward or against its own output patterns. To falsify these results:

1. **Use a different scorer model.** Run the same scoring prompt through GPT-4, Gemini, or Llama and compare scores. If the separation holds across models, it's measuring something real. If it collapses, it's measuring Claude-specific patterns.
2. **Use different slop generators.** The slop here was generated by Claude Haiku. Try GPT-4, Gemini, or human-written thought leadership. If the scorer can't separate human-written generic content from original reasoning, the metric needs work.
3. **Blind the scorer.** Remove all metadata and shuffle the samples. See if the scorer still clusters originals together and slop together without knowing which is which.
4. **Test on new originals.** Score blog posts from writers known for dense reasoning (Paul Graham, Gwern, Patrick McKenzie) alongside their topic-matched slop counterparts. If the scorer only works on the Vector Space posts, it's overfit.

The claim is: *structural reasoning density is computable, and surface humanization can't fake it.* Break it if you can.
